# Tutorial: Generate a DOCX with Equations from Scratch

This tutorial creates a DOCX file with `python-docx`, inserts text placeholders,
and replaces those placeholders with equations generated by `docx-equation`.
The same MathML source can produce native Word OMML output or MathType-compatible
output.

## 1. Install

```bash
python3 -m pip install docx-equation
```

For repository development:

```bash
python3 -m venv .venv
. .venv/bin/activate
python3 -m pip install -e ".[dev]"
```

`python-docx` is installed as a dependency of `docx-equation`.

## 2. Create a Base DOCX

Create the document with normal Word content and placeholder tokens. Keep each
placeholder in a single run so it can be replaced reliably.

```python
from docx import Document

base_docx = "tutorial_base.docx"

doc = Document()
doc.add_heading("Equation tutorial", level=1)

p = doc.add_paragraph()
p.add_run("Inline energy balance: ")
p.add_run("{{EQ_INLINE_001}}")
p.add_run(".")

doc.add_paragraph("Display equation:")
doc.add_paragraph().add_run("{{EQ_DISPLAY_001}}")

doc.save(base_docx)
```

## 3. Prepare MathML

Each equation is described by an `EquationSpec`. Inline equations omit the
`number` field. Display equations can include a number and use the shared
tabbed layout.

```python
from docx_equation import EquationSpec

inline_mathml = """
<math xmlns="http://www.w3.org/1998/Math/MathML">
  <mi>E</mi><mo>=</mo><mi>P</mi><mi>t</mi>
</math>
"""

display_mathml = """
<math xmlns="http://www.w3.org/1998/Math/MathML" display="block">
  <msubsup>
    <mi>P</mi>
    <mrow><mi>e</mi><mi>s</mi></mrow>
    <mrow><mi>c</mi><mi>h</mi></mrow>
  </msubsup>
  <mo>(</mo><mi>t</mi><mo>)</mo>
  <mo>=</mo>
  <mfrac>
    <mn>1</mn>
    <mrow><mn>1</mn><mo>+</mo><mi>s</mi><mi>T</mi></mrow>
  </mfrac>
</math>
"""

equations = [
    EquationSpec(
        placeholder="{{EQ_INLINE_001}}",
        mathml=inline_mathml,
        display=False,
    ),
    EquationSpec(
        placeholder="{{EQ_DISPLAY_001}}",
        mathml=display_mathml,
        display=True,
        number=1,
    ),
]
```

## 4. Export Native Word OMML

OMML output keeps equations editable with Word's native equation engine.

```python
from docx_equation import (
    EquationStyle,
    ExportOptions,
    NumberingOptions,
    OmmlOptions,
    embed_mathml_placeholders,
)

style = EquationStyle(
    font_family="Times New Roman",
    east_asia_font="SimSun",
    font_size_pt=10.5,
    number_font_size_pt=10.5,
)

numbering = NumberingOptions(
    chapter=1,
    number_format="(1SEP1)",
    separator="-",
    sequence_name="Eq",
)

summary = embed_mathml_placeholders(
    base_docx,
    "tutorial_omml.docx",
    equations,
    ExportOptions(
        target="omml",
        style=style,
        numbering=numbering,
        omml=OmmlOptions(font_family="Times New Roman", east_asia_font="SimSun"),
    ),
)

print(summary)
```

The display equation is centered with a tab stop, while the label is right
aligned. With the options above, the label is `(1-1)`.

Use a sequence-only label by changing the numbering options:

```python
NumberingOptions(number_format="(1)")
```

## 5. Export MathType-Compatible Objects

MathType output embeds OLE objects. The default `alternate-content` mode embeds
a MathType object and keeps an OMML fallback for Word environments without
MathType.

```python
from docx_equation import MathTypeOptions

summary = embed_mathml_placeholders(
    base_docx,
    "tutorial_mathtype.docx",
    equations,
    ExportOptions(
        target="mathtype",
        style=style,
        numbering=numbering,
        mathtype=MathTypeOptions(
            embed_mode="alternate-content",
            mathtype_version="DSMT4",
        ),
        omml=OmmlOptions(font_family="Times New Roman", east_asia_font="SimSun"),
    ),
)

print(summary)
```

Use `embed_mode="png-preview"` when the document should contain an explicit PNG
preview next to the embedded OLE object.

## 6. Complete Script

```python
from docx import Document

from docx_equation import (
    EquationSpec,
    EquationStyle,
    ExportOptions,
    MathTypeOptions,
    NumberingOptions,
    OmmlOptions,
    embed_mathml_placeholders,
)

base_docx = "tutorial_base.docx"

doc = Document()
doc.add_heading("Equation tutorial", level=1)
doc.add_paragraph("Inline energy balance: ").add_run("{{EQ_INLINE_001}}")
doc.add_paragraph("Display equation:")
doc.add_paragraph().add_run("{{EQ_DISPLAY_001}}")
doc.save(base_docx)

equations = [
    EquationSpec(
        placeholder="{{EQ_INLINE_001}}",
        mathml="<math xmlns='http://www.w3.org/1998/Math/MathML'><mi>E</mi><mo>=</mo><mi>P</mi><mi>t</mi></math>",
        display=False,
    ),
    EquationSpec(
        placeholder="{{EQ_DISPLAY_001}}",
        mathml="<math xmlns='http://www.w3.org/1998/Math/MathML'><mfrac><mi>a</mi><mi>b</mi></mfrac><mo>=</mo><mi>c</mi></math>",
        display=True,
        number=1,
    ),
]

style = EquationStyle(font_family="Times New Roman", east_asia_font="SimSun")
numbering = NumberingOptions(chapter=1, number_format="(1SEP1)", separator="-")

embed_mathml_placeholders(
    base_docx,
    "tutorial_omml.docx",
    equations,
    ExportOptions(target="omml", style=style, numbering=numbering),
)

embed_mathml_placeholders(
    base_docx,
    "tutorial_mathtype.docx",
    equations,
    ExportOptions(
        target="mathtype",
        style=style,
        numbering=numbering,
        mathtype=MathTypeOptions(embed_mode="alternate-content"),
        omml=OmmlOptions(font_family="Times New Roman", east_asia_font="SimSun"),
    ),
)
```
